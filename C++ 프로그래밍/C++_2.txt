Namespace란? 
=> 여러 개의 전역적인 라이브러리 함수 혹은 클래스 등이 중복되어 컴파일 상의
문제를 야기할 수 있으므로, 이들을 구분하여 중복되지 않도록 하는 키워드를 의미
어느 클래스에서 나온것인지, 어디소속인지 알려주기 위함.
예를 들어 std::cout 의 cout객체는 std라는 네임스페이스(iostream 클래스)의 소속이다 라는
것을 알려주면, cout이란 같은 이름이 와도 어느 클래스 소속인지 구분이 가능하므로 이렇게
네임스페이스를 명시해주는 것이다.

Namespace std의 구성

Namespace std {
	class ostream {
		:
	}
	class istream {
		:
	}
ostream cout;
istream cin;
} -> std::cout, std::cin을 써주어서 쓴다, ::를 쓰는이유는 클래스이름이 각각 : 이여서 두번써주는 것
ostrean,istream 클래스에 속하는 객체 cout, cin

using은 함수나 클래스 작성시 소속된 네임스페이스를 일일이 기술하는 작업이 번거로우므로 이들
네임스페이스를 생략하기 위해 using문 사용.
#include 아래에다가, using namespace std; 이렇게 정의를 함.
이러면 std가 현재소스에서 사용되고 있다라고 정의가 됨.

네임스페이스 => 클래스들끼리 구분하기위해 한 공간을 만들어 이름을 정의함. 네임스페이스만
호출해도 해당 클래스의 기능을 가져올 수 있다.

클래스 => 해당 클래스의 이름과 관련된 객체들을 만드는 공장이라고 할수있다.

객체 => 해당 클래스의 공장에서 찍어낸 결과물

예를들어 클래스 = 통조림 공장, 객체 = 통조림
